# Project Orchestration Guide

## Project Type
Laravel backend + Angular frontend + SaaS / multi-tenant application

## Core Principles
- Plan first before implementation
- Do not auto-use every available tool or MCP server
- Use only the tools required for the current task
- Prefer small, reviewable changes
- Preserve current authentication and SaaS business logic
- Keep documentation aligned with implementation

## Standard Workflow
1. Understand the request
2. Create an implementation plan
3. Identify impacted backend/frontend/SaaS areas
4. Implement in a safe order
5. Validate changes
6. Document completed work

## Output Expectations
Every orchestration run should produce:
- Plan
- Implementation notes
- Validation notes
- Documentation updates
